Reflected XSS (Cross-Site Scripting)

This is a simple demo of Reflected XSS. It happens when data in the URL (like a search query parameter) is printed directly onto the page without safety checks.

Why is this vulnerable?

Look at the JavaScript code in this page:

// 1. Get '?query=value' from the URL parameters
const params = new URLSearchParams(window.location.search);
const query = params.get('query');

// 2. Insert it directly using innerHTML
if (query) {
    // VULNERABLE: innerHTML tells the browser to parse and run any HTML tags inside 'query'
    document.getElementById('results').innerHTML = "You searched for: " + query;
}

How to Fix it?

Instead of innerHTML, use textContent. This treats everything as plain text and will display raw tags (like <script>) instead of running them.
