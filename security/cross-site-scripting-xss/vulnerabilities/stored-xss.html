Stored XSS (Persistent Cross-Site Scripting)

Stored XSS occurs when a website saves user input (like comments) in a database (here we simulate this with the browser's localStorage) and displays it later to other users without any safety check.

Why is this vulnerable?

When loading comments from the database, we insert them into the page using innerHTML: 

// 1. Loop through all comments in database
comments.forEach(comment => {
    const div = document.createElement('div');
    
    // VULNERABLE: Using innerHTML executes any script tags stored in the comment
    div.innerHTML = comment;
    
    commentsList.appendChild(div);
});

How to Fix it?

Instead of div.innerHTML = comment;, use div.textContent = comment; to safely treat the comment text as plain data.
