DOM-based XSS (Cross-Site Scripting)

DOM-based XSS happens entirely in the browser using client-side JavaScript. The script reads data from a Source (like the URL hash fragment #value) and writes it unsafely to a Sink (like innerHTML) without validation.

Why is this vulnerable?

The script reads the URL hash fragment (the Source) and puts it directly into the page using innerHTML (the Sink): 

// Source: location.hash
const hashValue = decodeURIComponent(window.location.hash.substring(1));

// Sink: innerHTML (Vulnerable)
document.getElementById('greeting').innerHTML = "Welcome " + hashValue;

How to Fix it?

Avoid using innerHTML when writing user-controlled parameters. Instead, use textContent to treat the string as plain text, preventing any HTML/JS execution.
