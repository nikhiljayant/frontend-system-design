Stored XSS Mitigation

This page shows the safe way to display persistent user data (like stored comments). Even if a malicious script gets saved into the database, it cannot execute because we render it safely.

Why is this secure?

Instead of injecting raw string text directly into innerHTML, we set the comment's value using textContent:

// SECURE render loop:
comments.forEach(comment => {
    const div = document.createElement('div');
    
    // SECURE: textContent instructs the browser to treat this purely as text
    div.textContent = comment;
    
    commentsList.appendChild(div);
});
