Why is this secure?

Look at the corrected JavaScript code in this page:

// 1. Get '?query=value' from the URL parameters
const params = new URLSearchParams(window.location.search);
const query = params.get('query');

// 2. Insert it safely using textContent
if (query) {
    // SECURE: textContent treats the input strictly as text, NOT executable HTML
    document.getElementById('results').textContent = "You searched for: " + query;
}

The Key Rule:

If you are inserting text that comes from a user, never use innerHTML. Always use textContent (or innerText). This is the simplest and most effective way to stop XSS.
