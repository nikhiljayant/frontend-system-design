DOM-based XSS Mitigation

This page shows how to securely read and render values directly in client-side JavaScript. By outputting hash data safely, we neutralize any script tags present in the URL hash.

Why is this secure?

The client-side JavaScript reads the hash value and writes it securely to the DOM using textContent:

// Source: location.hash
const hashValue = decodeURIComponent(window.location.hash.substring(1));

// Sink: textContent (Secure)
document.getElementById('greeting').textContent = "Welcome " + hashValue;

What is the rule?

Always output data from client-side sources (like location.hash, location.search, document.referrer) to safe DOM properties like textContent or innerText. If you must output HTML, use a dedicated HTML sanitizer library.
