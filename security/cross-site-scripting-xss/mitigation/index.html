How do we Mitigate XSS?

The golden rule of XSS defense is: Never trust user input. To prevent XSS, you must tell the browser to treat user data as data, not code:

  1. Use Safe APIs: In client-side JavaScript, use secure element properties like textContent or innerText instead of innerHTML or document.write.
  2. HTML Entity Encoding: If you must output data into HTML context, convert characters like < to &lt; and > to &gt; so the browser displays them instead of executing them as HTML tags.
  3. Sanitization: If you must allow formatted HTML comments (like bold or italic text), use a trusted HTML sanitization library (such as DOMPurify) to strip out dangerous tags and attributes before displaying them.
